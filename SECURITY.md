# 安全政策

## 支持版本

当前项目仍处于早期公开准备阶段。默认只维护 `main` 分支的最新代码。

## 报告漏洞

请不要在公开 Issue 中披露可利用细节。建议通过以下方式私下联系维护者：

- GitHub Security Advisory（仓库启用后推荐）
- 维护者在仓库主页公布的安全邮箱

报告时请尽量包含：

- 受影响版本或提交
- 复现步骤
- 影响范围
- 可行的缓解建议

## 安全边界

melonPaw 包含 Agent 工具执行、文件访问、Shell 调用、浏览器自动化、插件和技能加载等高风险能力。公开部署前应至少确认：

- 服务不要直接暴露到公网。
- API Key、模型密钥、渠道 token 和备份包不要提交到仓库。
- 工作区路径、文件访问、Shell 执行必须开启审批和隔离策略。
- 第三方插件、技能包和备份包只从可信来源导入。
